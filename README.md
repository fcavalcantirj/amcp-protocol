# AMCP: Agent Memory Continuity Protocol

> Cryptographic identity and verifiable memory for AI agents.

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

## The Problem

AI agents today suffer from:
- **Session amnesia** â€” Each conversation starts fresh
- **Platform lock-in** â€” Identity exists only within a specific service
- **Unverifiable claims** â€” "I remember our conversation" cannot be proven
- **Fork ambiguity** â€” When agents are copied, identity lineage is lost

## The Solution

AMCP gives agents:
- **Self-sovereign identity** â€” KERI-based cryptographic identifiers agents own
- **Verifiable memory** â€” Content-addressed checkpoints signed by agent keys
- **Capability delegation** â€” UCAN-based permissions that can be shared and attenuated
- **LLM-safe crypto** â€” Middleware that keeps keys away from the language model

## Design Principles

1. **Agents own their identity** â€” Not granted by platforms, generated by agents
2. **LLMs are untrusted** â€” Never see private keys, only opaque handles
3. **Capabilities over identity** â€” Authorization via UCAN, not ACLs
4. **Memory is content-addressed** â€” IPLD CIDs, Merkle-proofable
5. **Forks are branches** â€” New identity with lineage, not continuity break
